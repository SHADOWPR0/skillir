# Skill IR Specification v1alpha1

`skill.md` is the authoring surface. The compiler emits a JSON document with
`schema: urn:skill-ir:v1alpha1` and `kind: Skill`.

## Required metadata

```yaml
id: domain.skill-name
version: 0.1.0
title: Human-readable title
license: Apache-2.0
risk_class: low | medium | high | critical
maturity: reference | reviewed | robot_ir | sim_qualified | hardware_observed | cell_qualified | production_approved
profile: humanoid-bimanual
```

## Required sections

- `## Preconditions`
- `## Steps`, with numbered steps and an explicit `action:` mapping
- `## Success Criteria`
- `## Safe State` for high- and critical-risk work

Each step may define `action`, `observe`, `verify`, `on_failure`, and repeated
`evidence` references. The compiler refuses unknown primitives, missing robot
capabilities, unresolved actions, and non-measurable completion criteria.

## Separation of concerns

`InstructionIR` expresses the human procedure. `SkillIR` expresses portable,
robot-neutral physical intent. `TargetBundle` is generated by an adapter for a
specific embodiment, controller, simulator, or training pipeline. No source
document becomes direct motor control without an explicit target binding.
